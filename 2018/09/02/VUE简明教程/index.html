使用 watch 监听器来实时修正变量

条件渲染与循环渲染

v-if v-else-if v-else 分支流程控制

在 <template> 元素上使用 v-if 条件渲染分组，渲染结果将不包含 <template> 元素

v-if 会尝试复用以提高性能，这会导致输入框内的数据保留，若要标识不要复用，请设置不同的 key 属性

v-show也可以选择性显示和隐藏 v-show 不支持 <template> 元素，也不支持 v-else。

v-bind 绑定任意元素属性到data

v-bind 让任意属性绑定到data

v-bind:属性=”变量名” 一般简写为： :属性=”变量名”

在将 v-bind 用于 class 和 style 时，表达式结果的类型除了字符串之外，还可以是对象或数组。

v-on 绑定任意DOM事件到js代码

v-on 监听 DOM 事件，并在触发时运行 methods 中的函数或是一段 JavaScript 代码

v-on:click = “onclick” 可简写为：@click=”onclick”

@click=”onclick(12)” //传递参数

@click=”onclick(12,$event)” //特殊变量 $event 传递当前元素 在js中可 event.preventDefault() 来使用

修饰符来处理是vue推荐的做法:

事件修饰符、按键修饰符、.exact 修饰符、鼠标按钮修饰符

修饰符参见：VUE官方的修饰符说明

父子组件之间的通信

在 Vue 中，父子组件的关系可以总结为 prop 向下传递，事件向上传递。

父到子 prop

子组件显式使用 prop 声明它预期的数据、并在自己的内部正常使用（就像在data中一样）
js:
Vue.component(‘child’, {
props: [‘message’],
template: ‘‘
})
html:

注：HTML 不区分大小写 如果在prop中使用驼峰式，在HTML中使用短横线分割式

可以使用 v-bind 来传递动态

不应该在子组件中改变 prop ，prop在数值上是传递，在数组和对象上是引用

对于class和style来说，传入的值不会覆盖子组件的对应值，而是加空格合并字符串

可以为prop指定验证规则
